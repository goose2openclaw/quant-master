# QuantMaster Q@C 量化交易系统使用说明书

## 一、系统概述

### 1.1 什么是 QuantMaster Q@C？

QuantMaster Q@C (简称 Q@C) 是一款基于币安(Binance)交易所的**全自动量化交易系统**。系统采用多策略融合架构，通过V8信号、G12策略和MiroFish三个核心引擎协同工作，实现智能化的加密货币交易决策。

### 核心特点

- 全自动运行：无需人工干预，24/7持续监控和交易
- 三重策略引擎：融合多种量化策略，综合评分决策
- 十重风控保护：多层次风险控制，保障资金安全
- 实时监控：完善的日志和状态监控体系
- 自我进化：内置自我优化引擎，持续学习和改进

### 1.2 支持的交易所和交易对

**币安 (Binance)** - 主要支持的交易所
- 现货市场 (Spot)
- 合约市场 (Futures) - 部分功能

**支持的交易对包括但不限于：**
BTC, ETH, XRP, ADA, MATIC, SOL, DOGE, DOT, AVAX, LTC, LINK, UNI, ATOM, NEAR

## 二、核心架构

### 2.1 三重策略引擎

Q@C采用业界领先的三引擎协同架构：

#### V8 Signal Engine (50%权重)
V8信号引擎是系统的主要决策模块，负责：
- 实时技术指标计算（RSI、MACD、KDJ等）
- 多时间周期分析（1小时、4小时、日线）
- 市场情绪识别
- 动态权重调整

**技术指标包括：**
- RSI (相对强弱指数)
- MACD (移动平均收敛散度)
- Bollinger Bands (布林带)
- Volume (成交量)

#### G12 Strategy Engine (30%权重)
G12策略引擎专注于趋势跟踪和波段操作：
- 趋势识别和确认
- 支撑阻力位计算
- 突破交易策略
- 动态止损止盈

#### MiroFish Engine (20%权重)
MiroFish是独立运行的情绪和形态识别引擎：
- 币种形态识别
- 市场情绪分析
- 资金流向监控
- 异常波动检测

### 2.2 数据流程

市场数据进入DataBus数据总线后，由三个引擎并行处理：
- V8 Signal处理50%的权重
- G12 Strategy处理30%的权重
- MiroFish处理20%的权重

综合评分经过风控引擎检查后，执行或拒绝交易决策。

## 五、策略说明

### 5.1 交易信号生成

**综合评分计算：**
综合评分 = V8评分 × 0.5 + G12评分 × 0.3 + Miro评分 × 0.2

**信号阈值：**
- 强烈买入：综合评分 > 65，执行BUY
- 买入：综合评分 > 55，执行BUY
- 观望：45 < 综合评分 < 55，保持HOLD
- 卖出：综合评分 < 45，执行SELL
- 强烈卖出：综合评分 < 35，执行SELL

### 5.2 交易品种

系统默认监控的交易对及优先级：
- 比特币 (BTCUSDT) - 最高优先级
- 以太坊 (ETHUSDT) - 最高优先级
- XRP (XRPUSDT) - 高优先级
- 艾达币 (ADAUSDT) - 高优先级
- 狗狗币 (DOGEUSDT) - 普通优先级
- 索拉纳 (SOLUSDT) - 普通优先级
- 波卡 (DOTUSDT) - 普通优先级

## 六、风控机制

### 6.1 十重风控规则

#### 外部风控（ExternalRiskEngine）

1. CONCENTRATION_001 - 集中度风控
   触发条件：单币种占比超过40%
   执行动作：禁止买入，自动调仓

2. RSI_001 - RSI超卖保护
   触发条件：RSI低于30
   执行动作：禁止BUY，防止继续下跌

3. RSI_002 - RSI超买保护
   触发条件：RSI超过75
   执行动作：禁止SELL，防止踏空

4. RSI_003 - RSI极值止损
   触发条件：RSI低于20
   执行动作：触发止损程序

5. RSI_004 - RSI极值止盈
   触发条件：RSI超过70
   执行动作：触发止盈程序

6. VOL_001 - 波动率控制
   触发条件：波动率异常
   执行动作：降低仓位

7. TREND_001 - 趋势确认
   触发条件：逆势交易
   执行动作：拒绝信号

8. VOLUME_001 - 成交量确认
   触发条件：成交量萎缩
   执行动作：谨慎交易

9. SPEED_001 - 价格速度
   触发条件：急涨急跌
   执行动作：延迟决策

10. LIQ_001 - 流动性检查
    触发条件：流动性不足
    执行动作：拒绝交易

### 6.2 RSI保护机制详解

RSI (Relative Strength Index) 是最重要的技术指标之一：

- RSI计算周期：14
- RSI范围：0-100

RSI解读：
- RSI < 20：极度超卖，风险极高
- RSI < 30：超卖区域，谨慎买入
- RSI 30-50：中性偏弱
- RSI 50-70：中性偏强
- RSI > 70：超买区域，谨慎卖出
- RSI > 80：极度超买，风险极高

**v13 RSI保护规则：**
- RSI < 30：禁止执行买入
- RSI > 75：禁止执行卖出
- RSI < 20：启动止损程序
- RSI > 70：启动止盈程序

### 6.3 仓位管理

**动态仓位计算：**
仓位比例 = 基础仓位 × 信心系数 × 风险系数

信心系数：
- 高信心信号：1.2
- 普通信号：1.0
- 低信心信号：0.8

风险系数：
- 高波动市场：0.7
- 正常市场：1.0
- 低波动市场：1.3

## 八、故障排除

### 8.1 常见问题

#### 问题1：进程自动退出

原因：可能是网络波动或API调用失败

解决步骤：
1. 检查网络连接
2. 检查代理服务器
3. 查看日志中的错误信息
4. 重启进程

重启命令：
cd /home/goose/.openclaw/workspace/quant_master
nohup python3 -u qm/quant_master_qcv13_daemon.py >> qcv13_daemon.log 2>&1 &

#### 问题2：API请求超时

原因：网络延迟或代理问题

解决步骤：
1. 更换代理服务器
2. 检查网络稳定性
3. 调整超时参数

#### 问题3：订单无法成交

原因：市场流动性不足或价格偏离

解决步骤：
1. 检查订单簿深度
2. 调整滑点容忍度
3. 等待更好的入场时机

### 8.2 紧急停止

**立即停止所有交易：**
pkill -f quant_master_qcv13

### 8.3 数据恢复

如果需要恢复交易记录：
1. 登录币安官网
2. 进入订单历史
3. 导出CSV文件
4. 使用分析工具回溯
