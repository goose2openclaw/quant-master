### 3.2 迭代执行流程

```
迭代流程:
┌─────────────────────────────────────────────────────────────┐
│  1. 监测 → 2. 评估 → 3. 决策 → 4. 执行 → 5. 验证          │
└─────────────────────────────────────────────────────────────┘

1. 监测 (自动)
   ├── 市场数据流
   ├── 策略表现监控
   └── 竞品动态追踪

2. 评估 (AI判断)
   ├── 偏离度计算
   ├── 归因分析
   └── 迭代必要性

3. 决策 (规则+AI)
   ├── 是否迭代
   ├── 迭代优先级
   └── 迭代方式

4. 执行 (自动)
   ├── 参数调整
   ├── 策略替换
   └── 组合调整

5. 验证 (回测)
   ├── 历史验证
   ├── 模拟交易
   └── A/B测试
```

### 3.3 迭代层级

| 层级 | 频率 | 内容 | 自动化程度 |
|------|------|------|------------|
| L1 参数 | 每日 | 止盈/止损/阈值 | ✅ 全自动 |
| L2 策略 | 每周 | 信号权重/条件 | ⚙️ 半自动 |
| L3 组合 | 每月 | 权重配置 | 🔧 需确认 |
| L4 架构 | 每季 | 模块调整 | 📋 需审批 |
