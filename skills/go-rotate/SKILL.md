# go-rotate - 轮动策略技能

## 概述
go-rotate 是一款专注于板块轮动和币种轮动的策略技能，与撞球(pool)策略形成互补。轮动策略充分利用震荡行情，高抛低吸，在不放大量的情况下规避资金局限性。

## 与撞球(pool)策略的差异

| 维度 | 撞球(pool) | 轮动(rotate) |
|------|------------|---------------|
| **核心逻辑** | 资金像撞球一样连续进球 | 高抛低吸，板块轮转 |
| **适用市场** | 趋势市场 | 震荡市场 |
| **操作频率** | 中频(30秒扫描) | 高频(可10秒) |
| **资金需求** | 较高(30%仓位) | 较低(10-20%) |
| **放量要求** | 需要放量确认 | 无需放量和缩量皆可 |
| **持仓时间** | 中长(小时级) | 短到中期(分钟到小时) |
| **止损习惯** | 5%硬止损 | 3%紧凑止损 |
| **收益来源** | 趋势跟随 | 波动收益 |

## 核心概念

### 1. 板块轮动
不同币种板块在不同时期表现不同：
- **主流币**: BTC, ETH - 稳定但收益有限
- **DeFi币**: LINK, UNI, AAVE - 与ETH联动
- **Meme币**: PEPE, BONK, DOGE - 高波动
- **Layer2**: ARB, OP, MATIC - 轮动机会

### 2. 高抛低吸
```
价格 > 均值 + 标准差 → 高抛
价格 < 均值 - 标准差 → 低吸
价格回归均值 → 止盈
```

### 3. 缩量突破
- 无放量时的突破更可靠
- 缩量回调是买入机会
- 放量突破反而可能是陷阱

## 决策矩阵

### 市场环境 × 策略

| 市场环境 | 撞球 | 轮动 |
|---------|------|------|
| 趋势上涨 | ✅ 主用 | ❌ 辅助 |
| 趋势下跌 | ✅ 主用 | ❌ 辅助 |
| 震荡整理 | ❌ 辅助 | ✅ 主用 |
| 缩量横盘 | ❌ 禁用 | ✅ 主用 |
| 放量突破 | ✅ 确认 | ❌ 慎用 |

### 轮动阶段

| 阶段 | 信号 | 操作 |
|------|------|------|
| **低位** | 价格 < 均值 - 1.5σ | 买入信号 |
| **回归** | 价格向均值移动 | 持有 |
| **高位** | 价格 > 均值 + 1.5σ | 卖出信号 |
| **超买/超卖** | RSI < 30 / > 70 | 极端信号 |

## API

### 基本使用

```python
from go_rotate import RotationStrategy

rot = RotationStrategy()

# 获取轮动信号
signal = rot.get_signal('BTC')
if signal['action'] == 'buy':
    print(f"买入 {signal['symbol']} @ {signal['price']}")
    print(f"原因: {signal['reason']}")

# 获取板块轮动列表
sectors = rot.get_sector_rotation()
for sector, coins in sectors.items():
    print(f"{sector}: {coins}")

# 执行轮动
rot.execute_rotation()
```

### 参数说明

| 方法 | 说明 |
|------|------|
| `get_signal(symbol)` | 获取单个币种轮动信号 |
| `get_sector_rotation()` | 获取板块轮动列表 |
| `execute_rotation()` | 执行轮动交易 |
| `get_position_size()` | 获取建议仓位 |

3. 高位币→低位币 轮换
4. 单次轮动资金占比15%

## 与撞球策略的协同

```
撞球(pool) ──── 趋势市场 ────→ 持仓不动
    │                              │
    │                              ▼
    │                       趋势结束，切换
    │                              │
    ▼                              ▼
轮动(rotate) ←── 震荡市场 ←──────┘
```

### 协同规则

```
IF 撞球持仓进入"衰退(4)"阶段 AND 轮动信号出现 THEN
    执行撞球止盈 + 轮动建仓

IF 撞球无持仓 AND 市场进入震荡 THEN
    启用轮动策略

IF 轮动连续3次亏损 THEN
    降低轮动仓位50% + 等待撞球信号
```

## 与资产管理的双向直连

### 双向通信

```python
# 接收资产管理器的信号
class RotationStrategy:
    def __init__(self, asset_manager):
        self.asset = asset_manager
    
    def should_rotate(self):
        # 从资产管理器获取持仓状态
        positions = self.asset.get_positions()
        
        # 检查是否需要轮动
        for symbol, pos in positions.items():
            if self.is_overvalued(symbol):
                # 发送卖出信号给资产管理器
                self.asset.recommend_sell(symbol)
    
    def on_trade_executed(self, trade):
        # 通知资产管理器交易已执行
        self.asset.on_trade_done(trade)
```

### 信号交换

| 方向 | 信号类型 | 内容 |
|------|----------|------|
| 资产管理→轮动 | 持仓状态 | 当前持仓、成本、盈亏 |
| 轮动→资产管理 | 轮动建议 | 买入/卖出信号 |
| 轮动→资产管理 | 仓位调整 | 建议仓位变化 |
| 资产管理→轮动 | 资金状况 | 可用资金、风险阈值 |
