# Asset Monitor and Trader Skill

## 概述
解决账户丢失、无法自动恢复交易、无法自主币种调换的顽固问题。实现资产管理和自主币种调换的完全自动化。

## 核心问题与解决方案

### 问题1: 每次升级后账户缺失
**原因**: positions = {} 全局变量初始化时被重置
**解决**: 
- 启动时从Binance API同步所有账户持仓
- 状态持久化到磁盘
- 多账户联合查询

### 问题2: 不能自动回到实盘交易
**原因**: 缺少自动重启和状态恢复机制
**解决**:
- watch_dog监控进程状态
- 自动检测崩溃并重启
- 启动后自动恢复交易

### 问题3: 不能自动开启自动交易
**原因**: 没有自动启动逻辑
**解决**:
- 注册为系统服务
- 开机自动启动
- 异常退出自动重启

### 问题4: 币种之间不能自主决策调换
**原因**: 缺乏跨币种比较和预测机制
**解决**:
- Top10 Trader to Crypto信号输入
- 预测收益率比较
- 基于机会成本的调换决策

### 问题5: 总提示转入资金
**原因**: 没有使用现有资产的逻辑
**解决**:
- 默认不使用新资金
- 现有持仓中释放资金
- 卖出低预期收益币种

## 四大账户同步

### 1. 现货账户 (SPOT)
```
同步内容:
- USDT余额
- 所有持仓数量
- 持仓成本(通过平均买入价计算)
- 实时市价
```

### 2. 全仓杠杆 (CROSS MARGIN)
```
同步内容:
- 总净资产(BTC和USD)
- 各币种净额
- 借款情况
- 保证金率
```

### 3. 逐仓杠杆 (ISOLATED MARGIN)
```
同步内容:
- 各交易对持仓
- 基础货币和报价货币余额
- 杠杆倍数
- 强平价格
```

### 4. USDT合约 (FUTURES)
```
同步内容:
- 账户余额
- 持仓情况
- 未实现盈亏
- 维持保证金率
```

## 自主调换逻辑

### 决策流程
```
1. 扫描所有持仓币种
2. 获取Top10信号预测
3. 计算各币种预期收益
4. 比较机会成本
5. 决策:
   - 如果有币种预期收益 > 当前持仓最低收益 → 调换
   - 如果有币种进入启动/加速期 → 考虑买入
   - 如果有币种进入衰退期 → 考虑卖出
```

### 调换规则
```
规则1: 不引入新资金
规则2: 只用现有持仓释放资金
规则3: 调换前比较预期收益
规则4: 优先卖出衰退期币种
规则5: 优先买入启动/加速期币种
规则6: 最大持仓数限制(3个)
```

## 状态持久化

### 状态文件
```json
{
  "positions": {
    "BTC": {"entry": 70000, "amount": 0.01, "account": "spot"},
    "ETH": {"entry": 2000, "amount": 0.1, "account": "cross_margin"}
  },
  "last_trades": [...],
  "performance": {...},
  "last_sync": 1234567890
}
```

### 恢复流程
```
1. 加载磁盘状态
2. 查询API获取真实持仓
3. 对比并修正positions
4. 恢复交易状态
5. 继续扫描
```

## 使用方法

```python
from asset_monitor_and_trader import AssetMonitorTrader

amt = AssetMonitorTrader()

# 启动监控和交易
amt.run()

# 查询状态
status = amt.get_status()
print(f"总资产: {status['total']}")
print(f"持仓数: {status['positions']}")

# 强制同步
amt.force_sync()

# 手动调换
amt.switch_coin(from_coin="BTC", to_coin="ETH")
```
